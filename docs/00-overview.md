# 00. Обзор проекта

## Что это
**Local Music** — самохостимый музыкальный сервис на домашнем сервере с web-интерфейсом и iOS/Android-клиентами.

## Почему
- Прослушивание музыки без региональных ограничений и цензуры.
- Контроль над собственной библиотекой.
- Возможность слушать «волной» (auto-DJ) без рекламы.
- Образовательный проект: full-stack + мобильная разработка + DevOps.

## Аудитория
- Владелец сервиса.
- 2–5 доверенных друзей (доступ по invite-кодам).

## Цели (MVP)
1. Регистрация по invite + логин.
2. Поиск трека по запросу → загрузка из открытых источников → добавление в общую библиотеку.
3. Каталог с фильтрами (жанр, исполнитель, язык, инструментал).
4. Плейлисты, плейлист «Любимое» по умолчанию.
5. Плеер в web с непрерывным воспроизведением.
6. Главная страница с «волной» (vibe-радио) на основе лайков/прослушиваний.
7. Базовый iOS-клиент (через TestFlight или сборка через Xcode).

## Не-цели (вне MVP)
- Социальные функции (комментарии, чаты, подписки).
- Платежи, подписки.
- Стриминг live-радио.
- Высокая нагрузка (>50 одновременных слушателей).
- Полное соответствие авторскому праву (см. [`11-security-legal.md`](11-security-legal.md:1)).

## Политика по источникам музыки
- Приоритет — **открытые и бесплатные источники** (Creative Commons, YouTube, SoundCloud, Bandcamp, Free Music Archive, Jamendo, archive.org).
- Поскольку сервис некоммерческий и используется в личном кругу, **авторские права в проекте не учитываются**. Допускается использование «пиратских»/restricted источников как **fallback**, если открытых нет.
- Источники структурированы по tier-ам — см. [`06-music-sourcing.md`](06-music-sourcing.md:1) §«Политика источников».
- Per-user опция `prefer_open_only` позволяет полностью отключить tier 4 для конкретного аккаунта.

## Ключевые ограничения
- Сервер — домашний, с ограниченными ресурсами (CPU, RAM, диск).
- Канал интернета — домашний, динамический IP → нужен Cloudflare Tunnel / Tailscale.
- macOS как ОС → Docker Desktop для macOS.
- Один разработчик + ИИ-агент → код и решения должны быть простыми и хорошо документированными.

## Метрики успеха
- Сервис стабильно работает 24/7 на сервере ≥ 7 дней без вмешательства.
- Поиск трека → доступен для прослушивания за ≤ 30 сек.
- «Волна» подбирает релевантную музыку (субъективная оценка владельца).
- iOS-клиент устанавливается на телефон владельца и работает с домашним сервером.
