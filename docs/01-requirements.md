# 01. Требования

## 1. Функциональные требования

### 1.1 Авторизация и пользователи
- **FR-1.1** Регистрация только по invite-коду (env `INVITE_CODES`).
- **FR-1.2** Логин по email + пароль (или username + пароль).
- **FR-1.3** JWT access + refresh токены.
- **FR-1.4** Роли: `master` (владелец, единственный), `admin` (резерв), `user`.
- **FR-1.5** Master-аккаунт неудаляем, его статус неизменяем; редактирование других полей разрешено самому себе (см. [`15-master-account.md`](15-master-account.md:1)).
- **FR-1.6** Admin может: банить/удалять обычных пользователей, удалять треки. Master — всё то же плюс управление admin-ами и системные настройки.
- **FR-1.7** Смена пароля; восстановление пароля master — только через CLI на сервере.

### 1.2 Поиск и добавление музыки
- **FR-2.1** Пользователь вводит запрос (название трека / исполнитель / строка).
- **FR-2.2** Сервис ищет в внешних источниках (см. [`06-music-sourcing.md`](06-music-sourcing.md:1)) и возвращает список кандидатов (название, исполнитель, длительность, обложка, источник, preview-URL если есть).
- **FR-2.3** Пользователь выбирает кандидата → backend асинхронно скачивает, нормализует, извлекает метаданные, обогащает (MusicBrainz/Last.fm), сохраняет в библиотеку.
- **FR-2.4** Если трек уже в библиотеке (по хешу аудио или MBID) — возвращается существующий, не дублируется.
- **FR-2.5** Статус загрузки виден пользователю (websocket / polling).
- **FR-2.6** Альтернативный способ — пользователь сам загружает аудиофайл, указывает метаданные (название, исполнитель, альбом, обложка), backend выполняет верификацию достоверности (см. [`14-user-upload.md`](14-user-upload.md:1)).
- **FR-2.7** Если альбом не указан, backend сам пытается определить настоящий альбом через MusicBrainz (по mbid из AcoustID или fuzzy-поиску). Флаг сингла (`albums.is_single`) выставляется только тогда, когда трек действительно выходил как сингл; иначе создаётся обычный альбом (или используется уже существующий).
- **FR-2.8** Каждый добавленный трек получает запись в индексе источников (`track_sources`): откуда взят, URL/ID, кто добавил. См. [`16-library-index-and-restore.md`](16-library-index-and-restore.md:1).
- **FR-2.8.1** При ручной загрузке backend дополнительно ищет тот же трек в открытых источниках (YouTube/SoundCloud) и предлагает пользователю выбрать publishing-источник для индекса; если найдено — записи добавляются в `track_sources` с приоритетом выше, чем `user_upload`. См. [`14-user-upload.md`](14-user-upload.md:1) §3.
- **FR-2.9** Master может экспортировать индекс всей библиотеки (jsonl + манифест) и восстановить библиотеку на новом сервере командой `library restore`.
- **FR-2.10** Импорт треков и плейлистов из внешних сервисов (Spotify, Yandex Music, Apple Music, YouTube Music) — через OAuth-подключение или загрузку файла экспорта (CSV/JSON). См. [`17-external-imports.md`](17-external-imports.md:1).
- **FR-2.11** При импорте сначала ищется уже существующий локальный трек (ISRC/mbid/fuzzy), затем — автоматическая загрузка из открытых источников при `match ≥ 0.85`; неоднозначные треки попадают в очередь «выбрать вручную».

### 1.3 Каталог и фильтры
- **FR-3.1** Список всех треков библиотеки с пагинацией.
- **FR-3.2** Фильтры: жанр, исполнитель, альбом, язык (включая `instrumental`/`no-lyrics`), год, добавивший пользователь.
- **FR-3.3** Сортировки: по дате добавления, по популярности (число прослушиваний), по алфавиту.
- **FR-3.4** Поиск по библиотеке (full-text по названию/исполнителю/альбому).

### 1.4 Плейлисты
- **FR-4.1** При регистрации автоматически создаётся плейлист «Любимое» (нельзя удалить, можно переименовать).
- **FR-4.2** CRUD плейлистов; добавление/удаление треков; изменение порядка.
- **FR-4.3** Плейлист приватный по умолчанию; можно сделать видимым другим пользователям сервиса.
- **FR-4.4** Лайк трека = добавление в «Любимое».

### 1.5 Воспроизведение
- **FR-5.1** Стриминг аудио через HTTP range requests (поддержка перемотки).
- **FR-5.2** Очередь воспроизведения (queue) на клиенте.
- **FR-5.3** Запись фактов прослушивания (play events) для рекомендаций.
- **FR-5.4** Скробблинг (отметка прослушивания при >50% длительности).

### 1.6 Главная — «Моя волна»
- **FR-6.1** На главной — кнопка «Включить волну».
- **FR-6.2** Настройки волны: настроение (energetic/calm/sad/happy), жанр(ы), язык, «только знакомое» / «открывать новое».
- **FR-6.3** Алгоритм формирует бесконечную очередь рекомендаций (см. [`07-vibe-recommendations.md`](07-vibe-recommendations.md:1)).
- **FR-6.4** Кнопки «нравится / не нравится / пропустить» влияют на следующую рекомендацию.

### 1.7 Shazam (опционально)
- **FR-7.1** Распознавание трека по аудиофрагменту (см. [`08-shazam-integration.md`](08-shazam-integration.md:1)).
- **FR-7.2** После распознавания — автоматическое предложение добавить в библиотеку.

## 2. Нефункциональные требования

| Код | Требование |
|---|---|
| NFR-1 | Latency API: p95 < 300 ms (исключая загрузку аудио). |
| NFR-2 | Запуск всей системы одной командой: `docker compose up -d`. |
| NFR-3 | Хранение паролей: bcrypt/argon2. |
| NFR-4 | Все секреты — через `.env`, никогда не в git. |
| NFR-5 | Логи — структурированные (JSON), уровень настраивается. |
| NFR-6 | Бэкап БД + метаданных аудио — ежедневно (cron). |
| NFR-7 | Доступ извне дома — через Cloudflare Tunnel или Tailscale (без проброса портов). |
| NFR-8 | Покрытие тестами критичных модулей (auth, sourcing, recommendations) ≥ 60%. |
| NFR-9 | Документация по API генерируется автоматически (FastAPI → OpenAPI). |

## 3. User Stories (MVP)

| # | Как | Хочу | Чтобы |
|---|---|---|---|
| US-1 | новый пользователь | зарегистрироваться по invite-коду | получить доступ к сервису |
| US-2 | пользователь | вбить запрос «artist - track» | найти трек в интернете |
| US-3 | пользователь | выбрать из найденных кандидатов | добавить нужный трек в библиотеку |
| US-4 | пользователь | отфильтровать каталог по жанру и языку | найти музыку под настроение |
| US-5 | пользователь | добавить трек в плейлист «Любимое» | потом легко его найти |
| US-6 | пользователь | создать тематический плейлист | организовать музыку |
| US-7 | пользователь | включить «волну» с настроением «спокойное» | слушать фоном без выбора треков |
| US-8 | пользователь | поставить лайк/дизлайк во время волны | улучшить рекомендации |
| US-9 | владелец iPhone | установить iOS-приложение через TestFlight | слушать с телефона |
| US-10 | admin | удалить трек из библиотеки | очистить мусор |

## 4. Out of scope (для MVP)
- DRM, водяные знаки.
- Многоязычная локализация UI (только RU/EN).
- Полнотекстовый поиск по текстам песен.
- Эквалайзер, аудио-эффекты на сервере.

> Проект некоммерческий, для личного круга. Вопросы лицензирования контента и оплаты вне рамок проекта.
