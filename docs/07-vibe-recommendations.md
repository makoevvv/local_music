# 07. «Моя волна» — рекомендации

## Цель
Бесконечная очередь треков, подобранных под текущее настроение пользователя, с адаптацией к фидбеку (лайк / дизлайк / скип).

## Параметры сессии (vibe session)

```json
{
  "mood": "calm|energetic|sad|happy|focus|party|neutral",
  "genres": ["rock", "ambient"],
  "languages": ["en", "ru", "inst"],
  "mode": "familiar|discover|mixed",   // знакомое / новое / 50/50
  "explicit_allowed": true,
  "no_lyrics": false
}
```

## Модель «фичей трека»
Каждый трек получает вектор:
- `genres[]` (one-hot)
- `languages[]`
- `bpm` (нормализовано: tempo/200)
- `energy` (0..1)
- `valence` (0..1)
- `release_year_norm`
- `popularity` (внутренняя: log(play_count+1))

Если фичи отсутствуют — используем дефолты на основе жанра.

## «Профиль настроения» → фильтр и веса

| mood | bpm | energy | valence | hint genres |
|---|---|---|---|---|
| calm | 60-90 | <0.4 | 0.3-0.7 | ambient, jazz, classical, post-rock |
| energetic | 120-180 | >0.65 | >0.5 | rock, edm, hip-hop |
| sad | 60-100 | <0.5 | <0.4 | indie, acoustic, blues |
| happy | 100-140 | 0.5-0.8 | >0.65 | pop, funk |
| focus | 70-110 | 0.3-0.55 | 0.4-0.6 | lo-fi, instrumental, ambient |
| party | 110-140 | >0.7 | >0.55 | edm, hip-hop, pop |
| neutral | — | — | — | по любимым жанрам пользователя |

## Алгоритм (MVP — простой и понятный)

1. **Candidate pool:** треки, удовлетворяющие фильтрам (жанры/языки/настроение range).
2. **Score** каждого кандидата:
   ```
   score = w1 * mood_match
         + w2 * user_affinity      # сколько раз пользователь слушал/лайкал жанр/исполнителя
         + w3 * novelty            # если mode=discover — выше для несыгранных
         + w4 * popularity
         - w5 * recent_penalty     # если играл в последние N часов
         - w6 * dislike_penalty    # если дизлайк на трек/исполнителя
   ```
3. **Sampling:** не топ-K жадно, а Boltzmann-семплирование с температурой → разнообразие.
4. Возвращаем батч из 5 → пополняем по мере необходимости.

## User affinity (счётчики)
Пересчитываются периодически (раз в сутки фоновой задачей) и/или инкрементально:
- `user_artist_score[user, artist]` += играли (вес 1) / лайкнули (вес 3) / дизлайкнули (вес −5) / скип (вес −1).
- Аналогично для `user_genre_score`.

Хранение — в Redis (hash) для быстрого доступа, плюс снимок в БД.

## Обратная связь
- `like` → score трека/жанра/артиста ↑; следующий батч сильнее подстраивается.
- `dislike` → трек исключается из текущей сессии и downgrade артиста.
- `skip` в первые 15 сек → лёгкий минус.
- `played > 50%` → плюс к жанру/артисту.

## Холодный старт
Если у пользователя <20 прослушиваний:
1. Используем только фильтры настроения и явные жанры из пресета.
2. Подмешиваем популярные треки библиотеки.
3. Поощряем разнообразие (выше temperature).

## Будущее
- Эмбеддинги треков (audio2vec / OpenL3) → `pgvector` → cosine similarity.
- Коллаборативная фильтрация (если активных пользователей хотя бы 3-5).
- Контекст: время суток, день недели.
- A/B пресеты для самого владельца.

## Реализация (код)
- Модуль `app/services/vibe/`:
  - `mood_profiles.py` — таблица из секции выше.
  - `scorer.py` — функции скоринга.
  - `pool.py` — формирование candidate pool из БД (SQL + python).
  - `session.py` — управление сессией (Redis: `vibe:{session_id}:state`).
  - `feedback.py` — обработка реакций.
- Тесты: проверка, что фильтры/настроение корректно сужают пул, скоринг детерминирован при фиксированном seed.
