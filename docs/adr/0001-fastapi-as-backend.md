# ADR-0001: FastAPI как backend-фреймворк

- **Дата:** 2026-05-23
- **Статус:** accepted

## Контекст
Нужен backend на Python (предпочтение владельца). Требуется async I/O (стриминг, внешние HTTP-запросы к yt-dlp/MusicBrainz), автодокументация API для нескольких клиентов (web, iOS, Android), строгая типизация.

## Решение
Использовать **FastAPI** (0.110+).

## Альтернативы
- **Django + DRF** — мощный, но синхронный по умолчанию; больше boilerplate для нашего сценария; async поддержка пока неполная. Подходит для CRUD-сайтов, не для async-стриминга.
- **Flask** — минималистичный, но всё нужно собирать вручную (валидация, async, OpenAPI). Дольше путь.
- **Litestar** — современная альтернатива FastAPI, но меньше комьюнити и материалов.

## Последствия
- (+) Async из коробки, OpenAPI, Pydantic-валидация, dependency injection.
- (+) Огромное комьюнити, много примеров.
- (−) Меньше готовых батареек чем у Django (admin, auth) — будем писать своё.
- (−) Нужно выбирать ORM отдельно — выбрали SQLAlchemy 2 async.
