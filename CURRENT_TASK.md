# CURRENT_TASK — текущая задача для ИИ-агента

> **Как пользоваться:**
>
> - **Разработчик:** редактируешь этот файл — заполняешь секции под текущую задачу. Старые задачи можно либо переписывать поверх, либо архивировать в `docs/tasks-archive/` (опционально).
> - **Заполнять все секции необязательно.** Если непонятно, что писать в блок — оставь его пустым (или оставь только заголовок и плейсхолдер `—`). Минимум, который имеет смысл заполнить, — это «🎯 Что нужно сделать»; остальное опционально.
> - **ИИ-агент:** прочитай этот файл **первым** в каждом новом запросе. Если разработчик прислал короткое сообщение в чат (например, «продолжай», «делай»), всё равно опирайся на содержимое этого файла как на основной источник задачи. Если сообщение в чате противоречит файлу — уточни, что является источником истины.
>   - **Пустые секции — это нормально.** Не отказывайся работать только потому, что часть блоков пустая. Просто не делай предположений за разработчика по пустым секциям:
>     - Если пустая секция критична для выполнения задачи (например, нет «Критериев готовности» или «Что нужно сделать») — задай уточняющий вопрос.
>     - Если пустая секция не критична (например, «Открытые вопросы», «Ожидаемые артефакты», «Лог работы», «Ограничения») — просто пропусти её и работай по тому, что заполнено.
>   - При сомнениях лучше **спросить**, чем додумать.
>
> Если ты только что подключился к проекту впервые — сначала прочитай [`START_PROMPT.md`](START_PROMPT.md:1) и документы, на которые он ссылается, чтобы понимать контекст. Затем возвращайся сюда за конкретной задачей.

**Осталось (вручную, на стороне владельца — нужен GitHub-аккаунт):**

1. Создать пустой репозиторий на GitHub (например, `local-music`), **без** README/`.gitignore`/LICENSE (они уже есть локально).
2. Подключить remote и запушить:
   ```bash
   cd ~/Documents/local_music
   git remote add origin git@github.com:<your-username>/local-music.git
   # или https://github.com/<your-username>/local-music.git
   git push -u origin main
   ```
3. После пуша — отметить в [`docs/12-roadmap.md`](docs/12-roadmap.md:12) пункт «Создать GitHub-репозиторий и сделать первый push» как `[x]` и закрыть Этап 0.
